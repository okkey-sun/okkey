2. ノート（基本概念、文法、構文など）

📚 基本概念

シーケンス型
  • 順番のある要素の集まり(リスト、タプル、レンジ、バイナリシーケンス、文字列など)
関数の基礎
  • 引数: 関数に渡す値。自分自身を0番目として渡される
  • 戻り値(返り値): 関数を実行して得られる結果
  • 変数: 数値や文字列などの値を格納する「箱」。データを一時的に保管・管理するために使用
  • 注意点: 引数と引き渡す変数名が同名称でも引き渡される
ドット「.」の使い方 (参考サイト)
  • 変数 or プロパティ: ()がない
  • 関数(メソッド): ()がある
  • 実際にプロパティかどうかは定義を見ないと判断できない

🔢 データ型と構造

タプル
  • ()で囲むリスト型。要素を変更できない(不変)
辞書
  • KeyError: 存在しないキーを呼び出した場合に発生する例外
集合 (参考サイト)
  • a - b: 差集合(集合aの要素 - 集合bの要素)
  • a | b: 和集合(aとbのすべての要素)
  • a & b: 積集合(aかつbの共通部分)
  • a ^ b: 対称差(積集合を除く部分)
可変(mutable)と不変(immutable) (参考サイト)
  • 可変オブジェクト: 作成後も内部の要素を変更できる
    • list(リスト)、dict(辞書)、set(集合)
  • 不変オブジェクト: 作成後に内容を変更できない
    • str(文字列)、tuple(タプル)、int(整数型)、float(浮動小数点数型)
テンソル型 (参考サイト)
  • 機械学習時にデータを正規化するために、変数を学習用に割り当てるイメージ

⚠️ 例外処理とエラー (参考サイト)

assert文
  • デバッグ作業を容易にするために使用
  • 条件式がFalseの場合にAssertionErrorを発生させて停止
  • Trueの場合はそのまま処理が続く
  • if文との違い: assertはFalseの場合は例外を発生させる
except文
  • 例外処理を受け取る(キャッチする)
  • 想定外の処理が起きた時用の処理を用意
raise文
  • 例外を意図的に発生させたい場合に使用
  • 例外が発生する可能性のある箇所に記述
  • raiseで例外を発生させて、exceptでキャッチする
  • 例外の連鎖を作成するときに「from」をセットで使う
標準エラー(stderr)
  • sys.stderr: 標準エラーへ読み書きを行うためのファイルオブジェクト
  • プログラムの実行中に発生したエラーや警告などのメッセージを出力する場所
  • 通常、コンソールやターミナルに表示される
  • 標準出力とは別に出力先を分けることができる(例: 標準出力はファイル、標準エラーは画面)

📦 モジュールとファイル操作

インポート順序
  • from モジュール名 import 関数名
shutil モジュール
  • ファイルやディレクトリのコピー、移動、削除、圧縮・解凍など、高水準なファイル操作を行う標準ライブラリ
  • osモジュールよりも高レベルな機能を提供
  • ファイル管理、バックアップ、自動化スクリプトで非常に便利
JSONファイル操作
  • ファイルオブジェクトを扱う: json.load()、json.dump()
  • 文字列として扱う: json.loads()、json.dumps()(関数名に「s」が付く)
  • 違いは扱う対象が「文字列」か「ファイルやソケットオブジェクト」か
エンコーディング
  • コメントにおける「#」でエンコーディングを指示する場合、#の中で更に#を使ってエンコーディング宣言する

🔧 その他の重要ポイント

同名の優先順位と属性
  • インスタンスとクラスで名前がかぶった場合、インスタンスが優先
  • 理由: 深いところで上書きされるため
デフォルト引数
  • 変更できるリスト(可変オブジェクト)は基本的に使わない(問題集P68参照)
MAC、UNIX
  • sourceコマンドで使用する
GETとPOST
  • GET: 引数がアドレスで見えてしまうため、セキュリティに懸念がある
  • POST: 引数が見えないようにしているため、注意が必要
ER図(Entity-Relationship) (参考サイト)
  • カーディナリティとオプショナリティの記号:
    • || = 必須
    • ○ = オプション
    • - = 1つ
    • 3本(鳥足) = 複数

3. ノート（関数、メソッドなど）

基本的な出力と型変換

出力関連
  • end="" 改行しないで書く
  • sep="" 文章の空欄を埋める
エスケープシーケンス
  • \、\\、\'など通常の表現をキャンセルする
  • 改行が入るところで、\ を入れると改行がキャンセル
型変換関数
  • str(x) 文字列(String)型に変換
  • float(x) 浮動小数点に型を変換
  • int(x) 整数に型を変換

演算子

演算子一覧サイト

比較演算子
  • == 演算子は2つのオブジェクトが等価で同値であれば True を返す。等価で同じ値かどうか
  • != 演算子は左辺と右辺が異なる場合に True を返す
  • is は同じ値(==の結果がTrue)であっても、異なるオブジェクトであれば False を返す。2つのオブジェクトが同一かどうかを判定
算術演算子
  • += や *= 。例:x *= 0.8 は、xに0.8をかける
  • // 小数点以下を切り捨て
  • 文字列 * 数字は、数字が小数点ならエラー。整数ならその個数を並べる

リスト・シーケンス操作

基本関数・メソッド
  • len() 要素の数を調べる
  • dir() ()の中に"abc"や30(整数)、3.1などの小数点を入れると、使える関数が一覧で出る
  • del リストも関数なので引数である() で囲みそうだが、del[]が正解。命令文なのでこれだけ[]
  • list 他のデータをリストに変換する関数としても用いられる
  • insert(1, 4) 1番目に4を挿入。場所、要素の順
  • remove 最初の要素を削除
ソート・順序
  • sortはメソッド、sortedは関数で新しいリストを作成して返す
  • reversedで降順で表示するのが一番スマート
その他の便利な関数
  • range数字の範囲指定は、1つ前まで。例:(1, 21)は1から20まで。詳細
  • zip 2つ以上のシーケンスをペアで返したい場合に指定
  • enumerate()関数 要素の値だけでなく、(インデックス番号, 要素の値)とタプルで得られる

文字列操作

  • splitで分割、joinで連結
  • replace メソッドなので元々を変えないが、s=などで定義を変更
  • zfill 右寄せゼロ埋め。一番シンプルなのは文字列strのzfill()メソッドを使う方法

高度な関数

map と filter
  • map() イテラブル(リストやタプルなど)のすべての要素に組み込み関数やlambda(ラムダ式、無名関数)、defで定義した関数などを適用できる。リストなどの複数の要素に対して全てに指定した関数を適用させることができる関数
  • filter リストなどの複数の要素に対して条件が一致しないものを取り出すことができる関数
ジェネレータ
  • yield ジェネレータを使用してイテレータを作る時に、値を返すのはyield(returnではない)

制御構文

繰り返し(forやwhile構文)
  • continue 値がxの時はループに戻る
  • break 値がxになったらループを抜ける

例外処理

try構文
  • finally: try-except構文が終わる際に必ず実行される処理
  • else ブロック内でエラー(例外)が発生しなかった場合に、elseブロック内の処理が実行される
with構文
  • try…except…finallyと長くなるので、with構文で短縮
  • ファイルも自動的に閉じられるのが特色

モジュール

パッケージ管理
  • pip 複数のモジュールを組み合わせたパッケージをインストールする。例:QRコードなど
ファイル・パターンマッチング
  • glob ファイルパスのパターンマッチングで、主にファイルパスやファイル名
  • re 正規表現で文字列全体のパターンマッチングを行うためのモジュール。主に文字列全体
loggingモジュール
  • ログ出力なのか、ユーザ(プログラム実行者)に伝えたい情報としての出力なのかを切り分け
  • ログの種類をErrorやDebugのように、レベル分けが出来る
  • フォーマットを指定すれば、簡単に統一化された出力の仕方ができる
